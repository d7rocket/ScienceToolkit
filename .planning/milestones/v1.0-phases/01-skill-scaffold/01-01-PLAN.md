**File 1: `examples/output-sample.md`**

Create a complete fabricated sample output demonstrating the exact Instagram carousel format contract. This file is the canonical reference all downstream phases validate against.

Content requirements (all from locked CONTEXT.md decisions):
- Topic: James Webb Space Telescope discovery
- 6 slides total: hook (Slide 1) + 4 body slides (Slides 2-5) + takeaway/CTA (Slide 6)
- Metadata header: `# [Topic Title]` followed by `**Date:** YYYY-MM-DD | **Field:** [Field] | **Sources:** N`
- Horizontal rule (`---`) separates metadata from slides
- Each slide is an H2: `## Slide N: [Descriptive Title]`
- Slide 1 (hook): under 10 words, question or surprising fact. Emoji allowed.
- Body slides (2-5): ~150 chars each, 2-3 short sentences, punchy. Each ends with a cliff-hanger or question. NO emoji in body slides.
- Slide 6 (takeaway/CTA): key takeaway in one line + CTA like "Follow for daily science drops". Emoji allowed.
- Section order after slides: Caption, Hashtags, Sources, Images (each separated by `---`)
- Caption section: casual + authoritative tone, keyword in first sentence. CRITICAL: keep caption UNDER 2,100 characters total (not words). This is roughly 350-380 words max. The research flagged that 400-600 words would exceed Instagram's 2,200-char limit -- target the low end.
- Hashtags section: exactly 5 hashtags on one line, space-separated
- Sources section: numbered list with 3 sources demonstrating all citation variants:
  1. Published paper: authors, title, journal name (italicized), volume, year, DOI URL, source URL, `[Published in: Journal, Year]`
  2. Preprint: authors, title, arXiv (italicized), URL, `[Preprint - not peer reviewed]`
  3. News article: author, date, title, publication (italicized), URL, `[News article]`
- Images section: 1-2 placeholder image URLs

Use the exact citation examples from the RESEARCH.md code examples section for the Sources block (Gardner et al. 2023, Naidu et al. 2022, Overbye 2022).

**File 1: `.claude/skills/science/SKILL.md`**

Create the skill entrypoint that makes `/science [topic]` work in Claude Code.

YAML frontmatter (required fields):
```yaml
---
name: science
description: Generate an Instagram science carousel package for a science topic. Fetches source material, writes carousel slides, caption, hashtags, and citations.
argument-hint: [topic]
disable-model-invocation: true
allowed-tools: Read, Write, WebFetch, WebSearch, Bash
---
```

NOTE on path: Using `.claude/skills/science/` (NOT `skills/science/` from CONTEXT.md) per research recommendation -- this is the standard Claude Code auto-discovery path. CONTEXT.md likely used informal shorthand.

Markdown body -- step-by-step instructions Claude follows when `/science` is invoked:

**Step 1: Confirm topic**
- Show the user the topic (`$ARGUMENTS`) and the current date
- Ask them to confirm before proceeding
- If no topic provided, ask the user to provide one

**Step 2: Load grounding rules**
- Read `${CLAUDE_SKILL_DIR}/prompts/system.md`
- Apply all grounding rules for the remainder of the session

**Step 3: Load format reference**
- Read `examples/output-sample.md` (relative path from project root)
- This is the canonical format contract -- all output must match this structure exactly

**Step 4: Fetch source material**
- (Phase 1 scaffold: note that real fetching comes in Phase 2)
- For now, use placeholder sources clearly marked as fabricated
- In Phase 2+, this step will use WebFetch/WebSearch to gather real sources

**Step 5: Generate carousel output**
- Generate the full carousel package following the format in examples/output-sample.md
- Slide count: 5-7 slides
- Slide 1: hook under 10 words (question or surprising fact)
- Body slides: ~150 chars each, end with cliff-hanger or question
- Final slide: key takeaway + CTA
- Caption: under 2,100 characters, keyword in first sentence, casual + authoritative tone
- Exactly 5 hashtags
- Full APA citations with peer-review status labels
- At least one source image URL

**Step 6: Write output**
- Create slug from topic (lowercase, hyphens, no special chars)
- Write to `output/YYYY-MM-DD-[slug].md` using today's date

**Step 7: Print terminal summary**
- Print summary in this exact format:
```
Generating carousel for: [Topic]...
Generated: [Topic]
   Slides: N | Sources: N | Field: [Field]
   -> output/YYYY-MM-DD-[slug].md
```
- Do NOT print the full carousel content to terminal
