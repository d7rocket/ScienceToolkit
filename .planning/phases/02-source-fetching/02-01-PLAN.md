**Phase A: Parallel source search (all 6 channels simultaneously)**

Instruct the skill to perform these searches in parallel:

1. **arXiv** — `WebFetch https://export.arxiv.org/api/query?search_query=submittedDate:[{7_DAYS_AGO}0000+TO+{TODAY}2359]+AND+abs:{TOPIC}&start=0&max_results=5&sortBy=submittedDate&sortOrder=descending`
   - Date format: YYYYMMDD (GMT). Calculate 7 days ago from today.
   - Extract from each Atom entry: `<title>`, `<summary>` (abstract), `<published>` (parse year), `<author><name>` (first 3 + "et al."), `<id>` (extract arXiv ID), `<arxiv:journal_ref>` (if present), `<arxiv:doi>` (if present), `<link title="pdf">` href.
   - CRITICAL: Use `export.arxiv.org` domain exclusively — never `arxiv.org` (blocked by WebFetch bug #19287 per STATE.md).
   - Peer-review label: If `arxiv:journal_ref` is present: `[Published in: {journal_ref}]`. If absent: `[Preprint - not peer reviewed]`. Default to preprint — never falsely claim peer review.
   - No images from arXiv. `image_url: None`.

2. **PubMed** — Two-step fetch:
   - Step 1 ESearch: `WebFetch https://eutils.ncbi.nlm.nih.gov/entrez/eutils/esearch.fcgi?db=pubmed&term={TOPIC}&retmax=5&retmode=json&datetype=pdat&reldate=7&sort=pub+date`
   - Extract PMIDs from JSON response `esearchresult.idlist`.
   - Step 2 EFetch (top 2-3 PMIDs): `WebFetch https://eutils.ncbi.nlm.nih.gov/entrez/eutils/efetch.fcgi?db=pubmed&id={PMID1},{PMID2},{PMID3}&retmode=xml&rettype=abstract`
   - Extract: `<ArticleTitle>`, `<AbstractText>` (may have Label attributes), `<Journal><Title>`, `<PubDate><Year>`, `<Author>` list (LastName + ForeName, first 3 + "et al."), `<ArticleId IdType="doi">`, `<ArticleId IdType="pmc">`.
   - Label: Always `[Published in: {Journal}, {Year}]` — PubMed results are peer-reviewed by definition.
   - Image: If PMC ID present, note that full text with figures is at `https://pmc.ncbi.nlm.nih.gov/articles/PMC{id}/`. `image_license: [CC-licensed]` (PMC open-access figures are CC-licensed). If no PMC ID: `image_url: None`.
   - Rate limit note: Max 3 requests/second to NCBI. No API key needed.

3. **ScienceDaily** — Two-step:
   - Step 1: `WebFetch https://www.sciencedaily.com/rss/all.xml` — extract titles, links, descriptions, pubDates. Filter by last 7 days and topic relevance.
   - Step 2: `WebFetch {article_url}` for top 2-3 relevant articles. Extract body text, image URL (relative paths prefixed with `https://www.sciencedaily.com`), Journal Reference section with DOI.
   - Label: `[News article]`. Image license: `[Copyrighted - use with permission]`.

4. **Phys.org** — Two-step:
   - Step 1: `WebFetch https://phys.org/rss-feed/science-news/` — extract titles, links, descriptions, pubDates, `media:thumbnail` URLs.
   - Step 2: `WebFetch {article_url}` for top 2-3 relevant articles. Extract body text from article container, images from CDN (`https://scx1.b-cdn.net/csz/news/800a/{filename}`), DOI from citation section.
   - Label: `[News article]`. Image license: `[Copyrighted - use with permission]`.

5. **Nature News** — RSS + optional article fetch:
   - `WebFetch https://www.nature.com/nature.rss` — extract per item: title, link, `prism:doi`, `dc:date`, `dc:creator`, `content:encoded`.
   - DOI prefix routing: `d41586-*` = news/commentary -> `[News article]`; `s41586-*` = research -> `[Published in: Nature, {year}]`.
   - Filter `dc:date` within last 7 days.
   - Optionally fetch individual article pages for full body text. If article page fetch fails, use `content:encoded` from RSS as body.

6. **Ars Technica** — WebSearch fallback:
   - `WebSearch "site:arstechnica.com/science {TOPIC} {CURRENT_YEAR}"` to discover article URLs.
   - Attempt `WebFetch` on each discovered URL. If blocked, skip and log as unavailable.
   - Label: `[News article]`. Image license: `[Copyrighted - use with permission]`.

**Phase B: Quality gate**

After all fetches complete, apply the quality gate:
- Discard any source with body text under 2,000 characters UNLESS it is a PubMed abstract that will be paired with a news article covering the same finding.
- For PubMed abstracts that fall below the gate: measure abstract + all metadata fields together. If still short, keep as citation-only and pair with news.
- Log skipped sources in terminal output with reason.

**Phase C: Cross-source matching (FETCH-04)**

After quality gate:
- Compare titles and abstracts across all fetched sources. Group sources that cover the same finding.
- When a match is found: group the sources consecutively and add a note "Coverage of the same finding".
- Prioritize cross-validated findings (academic + news on same topic) when choosing what to feature.
- If no cross-match found after reviewing titles/abstracts, proceed immediately. Note "Sources not cross-validated" in output. Do NOT stall searching for matches.

**Phase D: Build structured source list for Step 5**

Compile all passing sources into a structured list. For each source, record:
- title, authors (first 3 + "et al." if more), journal/outlet name, year, DOI (or None), canonical URL, body text, peer-review label, image_url (absolute, or None), image_license label, source_type ("academic" or "news").
- Ensure at least 1 academic + 1 news source. Target 3-5 total.
- If a source channel returned 0 results for 7-day window, try relaxing to 30 days for that channel only. Log the relaxation.

**Failure handling:**
- Skip unreachable sources and log in terminal. Only fail the entire run if ALL sources are unreachable.
- Instruct to print which sources were searched, which returned results, and which were skipped.

**Remove the Phase 1 placeholder text entirely.** The Phase 1 exception text about fabricated sources must be deleted.

Do NOT modify Steps 1, 2, 3, 5, 6, or 7 in any way. Preserve all existing YAML frontmatter exactly.
