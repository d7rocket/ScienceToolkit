**Add a new useEffect** after the existing re-render useEffect. This effect attaches `editing:exited` listeners to IText objects on the canvas after each render:

```typescript
// Attach editing:exited handlers to IText objects for inline editing commit
useEffect(() => {
  const fc = fabricRef.current;
  if (!fc || !slides.length) return;

  const handlers: Array<() => void> = [];

  // Wait one frame for renderSlide to complete and objects to be on canvas
  const frameId = requestAnimationFrame(() => {
    const objects = fc.getObjects();
    for (const obj of objects) {
      if (obj instanceof IText) {
        const objName = (obj as { name?: string }).name;
        if (objName === 'title' || objName === 'body') {
          const handler = () => {
            const newText = obj.text ?? '';
            const currentSlide = useCarouselStore.getState().activeSlideIndex;
            if (objName === 'title') {
              useCarouselStore.getState().updateSlide(currentSlide, { title: newText });
            } else if (objName === 'body') {
              useCarouselStore.getState().updateSlide(currentSlide, { body: newText });
            }
          };
          obj.on('editing:exited', handler);
          handlers.push(() => obj.off('editing:exited', handler));
        }
      }
    }
  });

  return () => {
    cancelAnimationFrame(frameId);
    handlers.forEach(cleanup => cleanup());
  };
}, [slides, activeSlideIndex, colors, fontsReady, selectedFontPreset, alignmentOverrides]);
```

**Key implementation details:**
1. The handler reads `activeSlideIndex` from `useCarouselStore.getState()` (not from the closure) to get the current value at the time editing exits, not at the time the effect ran.
2. The handler calls `updateSlide(currentSlide, { title: newText })` or `updateSlide(currentSlide, { body: newText })` based on the object's `name` property.
3. `requestAnimationFrame` ensures the handler attaches after `renderSlide` has added objects to the canvas (renderSlide is called in a prior useEffect with the same deps).
4. Cleanup removes all `editing:exited` listeners and cancels the pending frame.
5. The CTA "Follow for daily science drops" text has `name: 'cta'` and is NOT handled — it is fixed copy.
6. The store update triggers a re-render cycle: `updateSlide` -> `slides` change -> useEffect fires -> `renderSlide` called -> new IText objects created. This is correct and safe because Fabric's `editing:exited` fires synchronously before React's re-render.

**IMPORTANT — Do NOT add a separate `canvas.on('text:editing:exited', ...)` listener.** Object-level listeners are more precise and avoid ambiguity about which object was edited.

**IMPORTANT — Do NOT call `renderSlide` from inside the `editing:exited` handler.** The store update triggers React's useEffect which calls renderSlide. Calling it directly would cause a double-render.
