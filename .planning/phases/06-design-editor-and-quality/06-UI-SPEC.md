Declared values (multiples of 4 only). Source: Tailwind v4 defaults confirmed from existing App.tsx usage (`p-6`, `gap-4`, `w-44`).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline dividers |
| sm | 8px | Swatch gap, font preset option padding |
| md | 16px | Panel section padding, control group spacing |

| xl | 32px | Between major panel sections |
| 2xl | 48px | — |
| 3xl | 64px | — |

Exceptions:
- Right sidebar inner padding: 16px horizontal, 24px vertical (`px-4 py-6`)

- `CONTENT_WIDTH`: 920px, `CONTENT_HEIGHT`: 810px

## Geometric Design Elements (QUAL-01 — default template quality)

Source: CONTEXT.md — "editorial with geometry" visual language, accent bar + corner mark.

### Body Slides
- Accent bar: 4px tall, 120px wide rectangle in `colors.accent` — positioned 16px below the title, left-aligned to CONTENT_X. Signals section break, creates visual anchor.
- Corner mark: 24x24px right-angle bracket shape (two 3px-wide Rect objects forming an "L") in `colors.accent` — positioned at top-right corner of the content area (CONTENT_X + CONTENT_WIDTH - 24, CONTENT_Y). Subtle geometric signature.

### Hook Slide
Visual treatment to differentiate from body slides (Claude's discretion — must feel like a distinct opener):
- Full-width accent band: `colors.highlight` colored Rect, 6px tall, spans CONTENT_X to CONTENT_X + CONTENT_WIDTH, positioned 48px above the title (top of safe zone area). Acts as a "topic introduction marker."
- Slide number / series indicator: "01" in `colors.accent` at 96px, weight 700, opacity 0.15, positioned centered behind the title as a ghosted watermark number. Creates depth without competing with title legibility.
- Title text-align: center (already implemented)
- Body subtitle: `colors.highlight` colored text (not primaryText), opacity 1.0 — distinguishes the teaser line from body slides

### CTA Slide
- Existing accent separator line (already implemented: 120px wide, 4px tall in `colors.accent`) — retained
- No new geometric additions (existing implementation is already visually distinct)

## Interaction Contracts

### Font Preset Picker (EDIT-01)
- Renders as a vertical stack of 4 buttons in the right sidebar
- Each button shows: preset name (16px semibold) + sample text "Aa" in the heading font at 20px + body font name at 12px
- Selected state: `border-[#6C5CE7]` border (2px solid), `bg-neutral-800` background
- Unselected state: `border-neutral-700` border (1px), `bg-neutral-900` background
- On click: update `selectedFontPreset` in Zustand store → triggers canvas re-render via existing useEffect subscriptions

### Color Editor (EDIT-02 + EDIT-03)
- Preset dropdown at panel section top: `<select>` element, full width, `bg-neutral-800` background, `text-white`, `border-neutral-700`
- 4 swatches in a 2x2 grid below the dropdown
- Each swatch: 32x32px filled square (CSS `background-color`) + label below at 12px weight 600 in `text-neutral-400`
- Swatch click: opens `<input type="color">` (browser native — no third-party library)
- Color input positioned off-screen (`position: absolute; opacity: 0`) triggered via `ref.current.click()` to avoid rendering the native widget inline
- On color change (`input` event): update `colors[role]` in Zustand store immediately → all canvases re-render via existing subscription
- Selected swatch outline: 2px solid `#6C5CE7` ring while color picker is open

### Inline Text Editing (EDIT-04)
- Double-click on title or body text on the main SlideCanvas enters edit mode
- Main SlideCanvas text objects switch to `IText` (single-line title) and `Textbox` (multi-line body) with `selectable: true, evented: true`
- Thumbnail canvases remain `selectable: false, evented: false` — no change to export source
- Edit mode cursor: default Fabric.js IText cursor (blinking line)
- Commit trigger: blur event OR Escape key → write updated text back to `slides[activeSlideIndex].title` or `.body` in Zustand store
- After commit: thumbnail re-renders automatically via `useEffect([slide, colors])` in Thumbnail component
- No explicit "save" button required — commit is implicit on blur/Escape
- No "reset to original" in this phase

### Text Alignment Toggle (EDIT-05)
- Renders in the right sidebar below color controls
- Three buttons: Left / Center / Right (icon-free — text labels "L", "C", "R" or arrows, whichever renders clearly at 14px)
- Use text labels: "Left" "Center" "Right" at 12px in a button group
- Applies to the active slide only
- Selected state: `bg-neutral-700 text-white`; unselected: `bg-neutral-900 text-neutral-400`
- On click: update alignment override for `activeSlideIndex` in Zustand store → canvas re-renders the active slide
- Alignment stored as per-slide override map in store: `Record<number, 'left' | 'center' | 'right'>` with default `'left'`

### Active Slide Indicator
- Selected thumbnail in ThumbnailStrip: 2px solid `#6C5CE7` ring around the thumbnail (existing `ring` behavior — confirm and lock to this color)
